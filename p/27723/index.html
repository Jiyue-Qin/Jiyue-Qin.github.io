Hexo博客搭建与GitHub部署

发表于 2020-03-24 分类于 博客 热度： ℃ Valine： 本文字数： 2.8k 阅读时长 ≈ 3 分钟

本文是介绍基于Hexo搭建的个人博客操作流程，通过上传至github实现公网访问。

用到Git，node.js以及VScode文件编辑软件等

1. 软件准备

软件介绍：

  • Git 属于分布式的版本控制工具，一般用于储存、追踪目录和文件的修改历史，优点是速度快，灵活，适合分布式开发，公共服务其压力和数据量小，大部分操作可以在本地完成不用联网，且能相对容易的解决冲突。

    • 在这里需要将Hexo博客工具安装到本地，上传到GitHub或Gitee，使用Git进行操作。

  • node.js： 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。

    • 这里使用的Hexo博客工具是基于Node.js开发的，需要安装相应环境进行本地操作。

官网下载地址：

  • Git：https://git-scm.com/downloads

  • node.js： https://nodejs.org/en/

    国外资源下载速度慢可以选择国内资源下载

软件安装注意事项：

这里使用的是Windows系统安装：

1) Git安装

  • 注意Git的安装路径不能出现中文与空格

  • 安装过程中除选择自定义安装路径外全部保持默认，进行下一步

  • 安装完成后启动cmd命令行工具，以查看版本号的形式验证Git是否已经安装成功

  • 安装成功后在空白处点击鼠标右键会出现两个新增右键选项：

    -> Git GUI Here

    -> Git Bush Here

    说明安装成功

    2) node.js安装：

  • 安装路径不要有中文

  • 保持默认值进行下一步到安装完成（so easy）

  • 安装完成后可以用Git Bush或cmd命令行窗口进行验证：

    • 输入node -v 显示版本号

2. 初始化博客

首先需要准备GIt与GitHub(Gitee)仓库

  • 详细操作参考：【轩枫阁】使用SSH密钥连接Github【图文教程】

1) 此处常见问题

有时会出现不能push到远端仓库的问题，可以先查看公钥绑定是否正确：

  • 打开文件夹：C:\Users\admin.ssh

  • 该文件夹有三个文件：

    • id_rsa
    • id_rsa.pub
    • known_hosts

    使用记事本打开 id_rsa.pub 文件，全选并复制（公钥，用于远端仓库与本地仓库的安全数据传输）

    打开gitee并登录，重新绑定公钥。

    参考连接里给出的是GitHub的操作，Gitee稍微不同。

    Gitee绑定公钥时应注意权限问题，错误信息下：
    [码云 push时提示 DeployKey does not support push code fatal: Could not read from remote repository.]

    (https://blog.csdn.net/soulwyb/article/details/90680676)

    正确操作：点击右上角的  添加个人公钥

    公钥配置好之后使用以下代码测试：

    ssh -T git@gitee.com

    请注意Gitee的Gitee Pages服务需要自己手动开启，开启请参考此处操作 : 【建站/Gitee】使用 Gitee 的 Gitee Pages 服务

5) 启动hexo

基本指令：

指令名 指令
启动 hexo s
新建博客 hexo n ‘博客标题’
生成静态文件 hexo g
清理缓存 hexo clean
启动本地服务 hexo s
部署 hexo d

详细指令参考官方文档：  https://hexo.io/docs/commands

启动后打开浏览器访问：  https://localhost:4000/
